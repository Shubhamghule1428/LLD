# Design Patterns

Design patterns are standardized, reusable solutions to recurring design problems.  
They provide **proven architectural structures**, not implementations.

This document classifies patterns based on **intent and system impact**.

---

# Pattern Classification

Design patterns are grouped into three categories based on the problem they solve:

- Creational Patterns  
- Structural Patterns  
- Behavioral Patterns  

Each category addresses a different dimension of system design.

----
# Pattern Category Comparison

| Category   | Focus              | Problem Solved               |
| ---------- | ------------------ | ---------------------------- |
| Creational | Object creation    | How objects are instantiated |
| Structural | Object composition | How objects are structured   |
| Behavioral | Object interaction | How objects communicate      |

----
# Design Perspective

- Creational patterns control **creation complexity**
- Structural patterns control **composition complexity**
- Behavioral patterns control **interaction complexity**
